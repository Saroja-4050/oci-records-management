Leveraging Generative AI to Enhance Information Request Management in Electronic Records

Oracle Database 26ai Converged Architecture

Overview

This system leverages Generative AI to enhance information request management in electronic records, providing an intelligent, end-to-end solution designed to automate and streamline Freedom of Information Act (FOIA) request processing. Built on Oracle Cloud Infrastructure's latest AI capabilities, this system leverages Oracle Database 26ai's converged architecture to provide unprecedented efficiency in document ingestion, intelligent retrieval, automated redaction, and compliance workflows.

How It Works: The system continuously collects unstructured data from fragmented sources (emails, repositories, file shares) at scheduled intervals, processes and indexes it proactively in Oracle Database 26ai. When a user submits a query, the system searches through this pre-indexed data using hybrid retrieval (vector + graph + keyword search), eliminating the need to process documents at query time.

Business Value

  • • 90% reduction in manual processing time
  • • Automated compliance with FOIA regulations
  • • Cost-effective serverless architecture
  • • Scalable to handle high-volume requests

Key Benefits

  • • Intelligent search via hybrid retrieval
  • • Automated redaction of sensitive information
  • • Complete audit trail for compliance
  • • Human-in-the-loop review for accuracy

Example Workflow

Phase 1: Continuous Data Collection & Indexing

Runs continuously or at scheduled intervals, collecting and indexing data from multiple sources before any user query arrives.

1
Collect Unstructured Data from Multiple Sources

OCI Data Integration and connectors pull data from fragmented sources:

  • • Email Systems: PST files from Outlook, Exchange servers
  • • Repositories: SharePoint, file shares, document management systems
  • • Fragmented Sources: Legacy systems, cloud storage, local drives
Example Sources:
• Email: city_council_emails_jan2024.pst (2.1 GB)
• Repository: \\fileserver\council\minutes\2024\january\
• SharePoint: /sites/citycouncil/Shared Documents/2024/
Collection Schedule: Daily at 2:00 AM UTC

Time: Continuous/Scheduled

2
Store Raw Data in Object Storage

All collected unstructured data is stored in OCI Object Storage immutable buckets:

Bucket: foia-raw-documents
Objects:
  • emails/2024/01/city_council_emails_jan2024.pst
  • repositories/2024/01/council_minutes_jan2024.pdf
  • sharepoint/2024/01/budget_discussions.pdf
Total Size: 15.7 GB (collected this cycle)
Timestamp: 2024-01-15T02:00:00Z

Time: ~5 minutes (for batch)

3
Document Understanding Processing

OCI Document Understanding processes all collected documents:

  • • OCR extracts text from scanned pages and images
  • • Parses PST files and extracts email content
  • • Identifies structured sections (agenda items, motions, votes)
  • • Extracts key-value pairs (dates, attendees, topics, senders)
  • • Detects handwritten annotations and signatures
Processing Output:
• 1,247 text chunks from all documents
• 342 structured sections identified
• 156 key-value pairs extracted
• 89 emails parsed from PST files
Confidence: 94.2% average OCR accuracy

Time: ~25 minutes (batch processing)

4
Indexing in Oracle Database 26ai Converged Store

All processed content is indexed in Converged Store for future queries:

Vector Search: 1,247 semantic embeddings created
Property Graph: 523 entities & 891 relationships extracted
SQL Metadata: 342 document records created
JSON Duality: Full document JSON stored
Keyword Index: Full-text search index created for exact matches

Time: ~12 minutes (batch indexing)

6
GenAI Agent Structures & Processes Query

The GenAI Agent receives the natural language query and structures it for multi-modal search:

Original Query: "Please provide all City Council meeting minutes from January 2024 that discuss budget allocations and personnel decisions."

Agent Processing:

  • • Extracts key concepts: "City Council", "January 2024", "budget allocations", "personnel decisions"
  • • Identifies date filter: January 2024
  • • Creates search strategy combining vector, graph, and keyword search
  • • Plans hybrid retrieval approach

Time: ~2 seconds

7
Hybrid Retrieval (Vector + Graph + Keyword)

The agent performs hybrid retrieval combining three search methods:

Vector Search:
Semantic similarity search:
• "Budget allocation discussion..." (0.92)
• "Personnel hiring decision..." (0.89)
• "Salary adjustments..." (0.85)
Results: 47 relevant chunks
Graph Traversal:
Relationship-based search:
• City Council → Budget Committee
• Personnel → HR Department
• Meeting → Agenda Items
Results: 23 related entities
Keyword Search:
Exact term matching:
• "budget allocations" (156 matches)
• "personnel decisions" (89 matches)
• "January 2024" (342 matches)
Results: 12 exact matches
Combined Results: Agent merges and ranks results from all three methods, identifying 8 most relevant documents spanning emails, PDFs, and meeting minutes.

Time: ~3 seconds
